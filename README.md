# Прогноз посещаемости культурных событий

Цель: построить модель машинного обучения, которая по историческим данным прогнозирует ожидаемую посещаемость/популярность событий (фестивали, выставки, мастер‑классы и т.п.).

## Данные
- Файл по умолчанию: `ai_event_dataset.xlsx` (поддерживаются также `.csv`).
- Можно загрузить несколько файлов и объединить их в один датасет.
- Целевая колонка определяется автоматически из списка или задаётся вручную.

## Возможности
- Автоопределение задачи (классификация/регрессия) и базовая предобработка.
- Несколько моделей: случайный лес, градиентный бустинг, гист. бустинг, линейная модель.
- Метрики качества, отчёт классификации, матрица ошибок.
- Прогноз для синтетического и пользовательского примера.
- Локальная LLM‑сводка результатов (Ollama).
- Автосохранение прогресса обучения в `training_runs/`.

## Подход
- Признаки делятся на числовые и категориальные. Пропуски заполняются медианой/модой, категориальные кодируются One‑Hot. Это делает данные пригодными для моделей sklearn и сохраняет максимум информации.
- Базовая модель по умолчанию — `rf` (RandomForest), так как она хорошо работает на табличных данных с нелинейностями и смешанными типами признаков. Для дисбаланса классов включён `class_weight="balanced"`.
- Доступны альтернативы (`gb`, `hgb`, `linear`) и подбор гиперпараметров через `--tune`.
- EDA‑графики и сводка сохраняются в `reports/`: `eda_summary.csv`, распределение целевой переменной и графики по числовым/категориальным признакам. Чтобы обновить, запустите `python train.py --data ai_event_dataset.xlsx` (EDA включён по умолчанию, отключить можно флагом `--no-eda`).

## Результат
- Пример метрик (запуск `training_runs/20260314_142305`): `accuracy = 0.45`, `f1_macro = 0.214` (см. `training_runs/20260314_142305/classification_report.json`, `training_runs/20260314_142305/confusion_matrix.csv`).
- Интерпретация: модель хорошо ловит средний класс (label `1`), но почти не распознаёт крайние классы (`0` и `2`), что видно по нулевому F1 для них. Поэтому `f1_macro` важнее, чем одна `accuracy`.
- План улучшений: балансировка классов (веса/ресэмплинг), расширение признаков, кросс‑валидация и тюнинг гиперпараметров.

## Запуск
1. Установка зависимостей:
   ```bash
   pip install -r requirements.txt
   ```
2. Десктоп‑приложение (без браузера):
   ```bash
   python desktop_app.py
   ```
3. CLI‑обучение (опционально):
   ```bash
   python train.py --data ai_event_dataset.xlsx
   ```
4. Веб‑интерфейс Streamlit (опционально):
   ```bash
   streamlit run app.py
   ```

## Локальная LLM (Ollama)
- По умолчанию используется `http://localhost:11434`.
- Модель берётся из `OLLAMA_MODEL` или выбирается первая доступная в Ollama.
- Переменные окружения: `OLLAMA_HOST`, `OLLAMA_MODEL`, `OLLAMA_TIMEOUT`.

## Примечание
OpenAI‑совместимые серверы поддерживаются в CLI и Streamlit через параметры LLM.
