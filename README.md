# Прогноз посещаемости культурно‑образовательных событий

Цель: построить модель машинного обучения, которая по историческим данным предсказывает вероятную посещаемость/популярность события (фестиваль, выставка, мастер‑класс и т.д.).

Данные: файл `ai_event_dataset.xlsx` (поддерживаются также `.csv`).  
Целевой признак определяется автоматически из списка:
- `Посещаемость_категория`
- `Доля_заполняемости_зала`
- `Посещаемость`

При необходимости можно явно указать `--target`.

**Подход**
1. Загрузка данных из Excel/CSV.
2. Очистка: приведение столбцов, обработка пропусков.
3. Предобработка: числовые признаки заполняются медианой, категориальные — модой и кодируются One‑Hot.
4. Разделение на train/test.
5. Обучение модели:
   - классификация: `RandomForestClassifier`
   - регрессия: `RandomForestRegressor`
6. Оценка качества:
   - классификация: Accuracy, F1‑macro
   - регрессия: MAE, RMSE, R²
7. Прогноз для синтетического «условного» события (мода категорий + медианы числовых).
8. Базовый EDA (гистограммы/бар‑чарты и сводная статистика) сохраняется в `reports/`.

**Запуск**
1. Установить зависимости:
```bash
pip install -r requirements.txt
```
2. Обучить модель и получить метрики:
```bash
python train.py --data ai_event_dataset.xlsx
```

**Полезные параметры**
```bash
# Явно указать целевой столбец
python train.py --data ai_event_dataset.xlsx --target Посещаемость_категория

# Запустить кросс‑валидацию
python train.py --data ai_event_dataset.xlsx --cv 5

# Сохранить обученную модель
python train.py --data ai_event_dataset.xlsx --save-model models/model.joblib

# Отключить EDA
python train.py --data ai_event_dataset.xlsx --no-eda
```

**Выход**
- Метрики качества на тестовой выборке.
- Предсказание для синтетического примера.
- Файлы EDA в `reports/` (`eda_summary.csv`, графики распределений).
